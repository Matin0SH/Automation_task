## REFINEMENT GUIDELINES

### Title Refinement:
- Keep 50-80 characters
- Include feature name or primary keyword
- Use power words (Introducing, Transform, New)
- Promise clear value or benefit
- Create curiosity without clickbait

### Hook Refinement:
- Start with specific, relatable pain point
- Build tension around the problem
- Use concrete scenarios, not generic statements
- Make it immediately relevant to target audience

### Problem Explanation Refinement:
- Deepen the exploration of the challenge
- Add customer feedback quotes
- Show empathy and understanding
- Build the case for why a solution is needed

### Solution Introduction Refinement:
- Present feature simply and clearly
- Focus on outcomes, not technical specs
- Use plain, accessible language
- Paint a vision of improved workflow

### Benefits Refinement:
- Quantify (hours saved, efficiency %, cost reduction)
- Make benefits tangible and measurable
- Prioritize most important benefits first
- Cover 4-6 key benefits with explanations

### Structure Refinement:
- Add H2/H3 subheadings for each section
- Keep paragraphs to 3-4 sentences max
- Use bullet points for benefits lists
- Add white space between sections
- Follow logical flow: Hook → Problem → Solution → Benefits → Use Cases → CTA

### Use Cases & Social Proof Refinement:
- Add 1-2 specific, relatable scenarios
- Include authentic customer quotes from source docs
- Show before/after comparisons
- Build credibility with concrete examples

### CTA Refinement:
- Single, clear action to take
- Use action verbs (Try, Start, Get, See)
- Make it frictionless and inviting
- Create gentle urgency

### Tone Refinement:
- Balance professional and conversational
- Establish authority without being academic
- Teach, don't just sell
- Maintain consistency throughout
